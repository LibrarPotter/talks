##What is Jekyll?
### 1) Jekyll is a: + Static Blog Generator + Markdown and liquid tag converter + The software behind GitHub pages + Wicked cool tool
### 2) Jekyll is also + written in Ruby (installed as a gem locally) + free and open sources + supported by GitHub
### 3) Jekyll's Background Written by [Tom Preston-Werner](https://en.wikipedia.org/wiki/Tom_Preston-Werner) in 2008. While Jekyll is used to create websites and static blogs, it's actually more of a parsing engine. You provide Jekyll the content, it outputs a static website (no database required).
## Getting Started with Jekyll (Mac and Linux)
### 1) Install Ruby Run ``` ruby --version``` anything above 2.0 is good. No Ruby? Check out the [Ruby documentation](https://www.ruby-lang.org/en/downloads/)
### 2) Install RubyGems RubyGems is a package manager for Ruby. [Download it and install it](https://rubygems.org/pages/download). Run ```gem -v``` to make sure you have it.
### 3) Optional - Install Bundler This is not required but highly recommended by GitHub. Bundler assists with tracking gems and versions. Run ```gem install bundler```
### 4) Create a Gemfile In your repository create a ```Gemfile``` ([example](https://github.com/bulib/studyGroup/blob/gh-pages/Gemfile)) Your Gemfile should read ``` source 'https://rubygems.org' gem 'github-pages' ```
### 5) Install GitHub Pages Gem ```bundle exec jekyll build --safe```
### 6) Running your site locally My method ``` bundle exec jekyll serve ``` GitHub's [new method](https://help.github.com/articles/setting-up-your-pages-site-locally-with-jekyll/) (I can't get this to work) ``` bundle exec jekyll build --safe jekyll serve ```
### 7) Updating ``` gem update jekyll ```
### 8) A few other tips Use ``` gem update ``` to update all gems Use ``` gem clenup ``` to clean up your gems Use ``` gem install kramdown``` to install kramdown

## Running Jekyll
### Basic Commands (GitHub version) Serve local website: ```bundle exec jekyll serve``` Build website: ```bundle exec jekyll build```
### Basic Commands (Jekyll version) Build website: ```jekyll build``` Watch for changes: ```jekyll build --watch``` Serve website and watch: ```jekyll serve``` Serve without watching: ```jekyll serve --detach```
### List of Commands Can be [found in Jekyll's documentation](http://jekyllrb.com/docs/usage/).
## Basic Blog Structure See the [official documentation](http://jekyllrb.com/docs/structure/) for more detailed information. Below is a summary.
### _config.yml This file has your configuration information. See [documentation](http://jekyllrb.com/docs/configuration/) for more information.
### _drafts A folder for unpublished content. These files should be labeled without a date.
### _includes A folder for "partials" that can be added to pages. Examples are header.html or footer.html. Syntax for including files is: ``` {% include file.ext %} ```
### _layouts These are your templates for laying out posts (selected in the YAML front matter). Use ```{{ content }}``` as a placeholder for post content
### _posts This is your content. The naming convention is ```YEAR-MONTH-DAY-title.markup```. You can customize the permalink for each post but the date and markup are determined by the file name.
### _data This folder is for your site data. Files should be ```.yml```, ```.yaml```, ```.json```, or ```.csv```. Data is accessible via ```site.data```.
### _site This is where your site will be!
### .jekyll-metadata A metadata file Jekyll uses to track which files have been updated since the last build.
### _index.html If it has YAML Front Matter it will be transformed by Jekyll. This is also true for any ```.html```, ```.markdown```, ```.md```, or ```.textfile```.
### Other files Everything else like ```css``` and ```images``` will be copied over to your site.

## Variables In Jekyll there are 3 types of general variables: global, site, and page. These variables can be accessed using ```{{ page.VAR }}```
### Predefined Global Variables ```layout``` if set uses the layout file name without the extension ```permalink``` use this to set your the final post url (assuming you don't like the /year/month/day/title.html default) ```published``` set to false if you don't want a post to show up
### Predefined Global Variables ```category``` set in the post's front matter (use a comma separated or YAML list). ```tags``` similar to category (set in same manner and added to front matter) but represents a tag.
### Site Variables For the [complete list see the documentation](http://jekyllrb.com/docs/variables/). I'll cover a few here. ```site.pages``` list of all pages
### Site Variables ```site.posts``` list of all posts ```site.categories.category``` list of all posts in a category ```site.tags.tag``` list of all posts with a tag
### Page Variables For the [complete list see the documentation](http://jekyllrb.com/docs/variables/). I'll cover a few here. ```page.title``` page's title ```page.tag``` page's tags ```page.category``` page's category.
## Notes on Liquid
### What is Liquid [Liquid is an open source](https://docs.shopify.com/themes/liquid) template engine that's written in Ruby and was developed by Shopify.
### Syntax - Basic Liquid has some fun syntax and uses ```{{ }}``` and ```{% %}``` for tags and objects. See the [documentation for complete examples](https://docs.shopify.com/themes/liquid/basics).
### Syntax Example - If Statement ``` {% if post.title == 'example' %} An example {% endif %} ```
### Syntax Example - Variables ``` {{ post.title }} ``` prints the post's title
### Syntax Example - For Loop ``` {% for post in site.posts %} {{ post.title }} {% endfor %} ```
### True and False ```{% if var == true %}``` will always be true if var is assigned Empty strings are true (check to see if they are blank) [Read the docs](https://docs.shopify.com/themes/liquid/basics/true-and-false) for more information
